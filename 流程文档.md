## 业务流程
1. 先进入 `汇总表` 阅读 `定制须知&售后条款`，确认授权、内容限制、回复节奏、售后边界。
2. 根据需求选择 `单走制品` 或 `拼车制品`。单走更像独立报价/独立交期；拼车通常有单坑数量、满坑/成车、色号由车头决定等规则。
3. 进入对应品类 Tab，读取文件格式、色彩模式、文件命名、邮件标题、交稿邮箱、正文收货信息、是否带肾图、打样/大货规则。
4. 按规格和数量查询价格矩阵：尺寸、单面/双面、烫单色/烫双色、触感银、板材、包边、底座、挂件、加购等会影响价格。
5. 如 Tab 内有二级链接，必须进入二级页面查看工艺规范、闪底图册、色号、白墨、车况或云母板常备色号；这些资料已保存为截图、图片资产和 OCR。
6. 系统落地时，提交订单前应校验：品类是否可做、规格是否在报价范围、数量是否满足起订/成车、文件命名和邮箱是否匹配、是否需要二级参考资料字段。

## 建议的数据模型
| 表/模块 | 主要字段 | 来源 |
|---|---|---|
| `categories` | 品类名、单走/拼车、是否可打样、默认交期、负责人/客服 | `汇总表` + 各 Tab 顶部说明 |
| `price_rules` | 品类、规格、尺寸、数量阶梯、工艺、单价、打样价、是否包邮、起订量 | 各 Tab CSV 的报价矩阵 |
| `submission_rules` | 文件名模板、邮件标题模板、交稿邮箱、正文要求、色彩模式、文件格式、DPI、图层规则 | 各 Tab 前 10 行 + 工艺规范 OCR |
| `process_specs` | 工艺类型、规范截图、OCR 文本、适用品类、风险/售后规则 | 二级 doc 截图与 OCR |
| `media_assets` | 图片路径、来源页面、来源单元格/截图、OCR、标签、可下载 URL | `image_library.html` + `media_library.json` |
| `external_refs` | 标题、URL、类型、本地目录、资产数量、刷新时间 | `secondary_pages.json` |
| `order_validation` | 所选品类、规格、数量、色号/闪底、文件命名、邮箱、是否满足成车/起订 | 上述规则组合 |

## 后续系统化的最小闭环
1. 先把 `汇总表` 的单走/拼车分类和 10 个品类 Tab 入库。
2. 每个品类先做一个“报价查询页”：输入尺寸、数量、工艺，返回价格和打样/大货规则。
3. 增加“交稿检查页”：根据品类生成文件名模板、邮箱、图层/色彩/DPI 要求。
4. 增加“素材选择页”：闪底、冷烫膜色号、云母板色号、工艺规范图片从 `media_assets` 读取。
5. 最后做订单流：选择品类 -> 选规格/工艺 -> 算价 -> 交稿检查 -> 生成邮件/订单摘要。
